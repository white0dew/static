# 个人名片入口页改版设计

## 1. 背景与目标

当前站点是一个以文章列表为核心的静态博客首页，首屏强调文章分发，不符合“个人名片入口”的目标。

本次改版目标：

- 将首页重构为单屏优先的个人名片入口页
- 参考 `https://mitbunny.ai/#` 的信息组织方式，但不复制其品牌、文案、配色和人物气质
- 视觉方向调整为更偏男性化、克制、冷静、硬朗的个人品牌页
- 所有外链先使用 mock 链接，占位即可
- 必须兼容移动端，优先保证手机首屏完整性和点击可用性

非目标：

- 不在本次改版中扩展新的文章系统
- 不引入框架、构建工具或后端服务
- 不做复杂多语言系统，仅保留必要的占位入口

## 2. 设计原则

- 入口优先：打开页面后先认识“人”，再看到可点击入口
- 信息收敛：首页不再展示多篇文章卡片流
- 单页完成：主要信息在一个页面中闭环，不依赖深层导航
- 中性偏男性：避免粉色、甜美、软萌风格，采用冷色与深色对比
- 移动端优先：手机端导航、头像区、入口卡片都必须在窄屏下自然堆叠

## 3. 用户体验目标

用户进入首页后，应在 5 秒内得到以下信息：

1. 这是哪个人的页面
2. 这个人主要做什么
3. 有哪些主要入口可以继续点击
4. 如何联系或关注

成功标准：

- 首屏不出现“博客列表首页”的既视感
- 主入口卡片清晰可见，至少 4 到 6 个
- 手机端不出现导航拥挤、横向溢出、按钮难点的问题
- 页面整体气质更像“个人品牌名片页”，而不是“内容站首页”

## 4. 信息架构

首页改为以下结构，自上而下组织：

### 4.1 顶部轻量栏

- 左侧：品牌名或个人名
- 右侧：少量辅助入口
- 不保留原有“文章 / 关于我 / 联系 / 博客”博客式导航

建议保留内容：

- `中文 / EN` 占位切换按钮，或一个轻量外链按钮

### 4.2 核心名片区

核心名片区位于视觉中心，包含：

- 头像
- 中文名
- 英文名或昵称占位
- 一句话身份标签
- 一段简短自我介绍

文案语气要求：

- 简洁直接
- 避免“博主感”与“公众号说明书感”
- 更像个人品牌介绍，而不是博客副标题

### 4.3 联系方式区

以两列或自适应网格展示基础联系信息：

- 微信
- 邮箱
- 城市
- 博客
- GitHub
- 其他社交平台占位

要求：

- 每项都支持替换为真实链接或文本
- 未填写真实值时使用 mock 内容
- 手机端自动折叠为单列或双列紧凑布局

### 4.4 主入口卡片区

这是首页的核心交互区，采用纵向堆叠的大卡片按钮。

初始放置 6 个入口：

1. 个人博客
2. 精选项目
3. 作品集
4. 关于我
5. 联系方式
6. 文章归档

要求：

- 卡片要明显强于普通文本链接
- 每张卡片包含标题、简短说明、箭头或方向性图标
- 可全部使用 mock 链接
- 允许后续手动替换为真实 URL

### 4.5 底部页脚

只保留轻量版权或备案信息。

要求：

- 弱化存在感
- 不再让页脚承担主要信息展示职责

## 5. 视觉设计

### 5.1 视觉方向

整体气质参考“居中人物 + 入口卡片”的结构，但视觉改为：

- 背景：冷灰蓝、石墨灰、雾白的柔和渐变
- 前景卡片：深色实体卡或深灰玻璃卡
- 文字：高对比、清晰、偏锋利的粗细关系
- 氛围：克制、现代、有一点科技感，但不做赛博炫技

### 5.2 与参考站的差异化要求

- 不使用粉色主色
- 不使用原站类似的甜美头像氛围
- 不复用原站文案结构与文案语气
- 不模拟其品牌名称、产品名称和内容项
- 只借鉴“名片 + 入口列表 + 邮件订阅位可选”的信息骨架

### 5.3 字体与层级

- 使用现有系统字体栈的前提下，强化字号和字重层级
- 主标题更强，副标题更短，辅助信息更淡
- 按钮和卡片标题要具备明显点击感

### 5.4 动效

仅保留轻量动效：

- 首屏元素渐进出现
- 卡片 hover 浮起
- 按钮或箭头轻微位移

禁止：

- 大范围炫技动画
- 影响可读性的背景运动

## 6. 响应式设计

### 6.1 桌面端

- 页面整体内容居中
- 名片区宽度控制在舒适阅读范围
- 联系方式区可双列显示
- 主入口卡片采用单列纵向排列

### 6.2 移动端

重点要求：

- 顶部区域不能再出现当前版本的拥挤换行问题
- 头像、标题、自我介绍、联系信息、入口卡片按垂直顺序自然排布
- 按钮高度和间距满足手指点击
- 文案长度控制，避免首屏被大段文字撑爆

建议断点：

- `<= 768px` 进入移动优化
- `<= 480px` 进一步压缩边距、字号与卡片间距

## 7. 技术设计

### 7.1 文件范围

优先修改以下文件：

- `index.html`
- `css/style.css`
- `js/main.js`

不要求修改文章页模板，除非共享样式被首页重构影响。

### 7.2 HTML 结构调整

首页将从“博客首页结构”改为“个人名片结构”：

- 删除首页文章筛选区和文章卡片区
- 删除“关于我 / 联系”作为独立长区块的旧布局
- 新增居中主卡片区域
- 新增联系信息网格
- 新增主入口按钮列表
- 可保留语义化 section，但要收敛成更少、更完整的模块

### 7.3 CSS 策略

- 复用一部分现有变量命名，但整体重新定义色板和阴影
- 强化卡片组件和按钮组件的一致性
- 清理不再使用的博客列表、文章筛选相关样式
- 保持首页和文章详情页样式互不误伤

### 7.4 JS 策略

首页移除文章筛选逻辑。

保留或重写的最小脚本包括：

- 年份自动更新
- 首屏 reveal 动画（如果仍需要）
- 可选的复制联系方式、平滑滚动等轻量交互

要求：

- 不保留无用脚本
- 不引入额外依赖

## 8. 错误处理与可维护性

- 所有 mock 链接使用统一占位值，便于后续批量替换
- 对头像缺失、链接缺失等情况提供稳定默认展示
- JS 对不存在元素保持安全判空
- 清理无用 DOM 与无用样式，避免首页后续继续向“博客首页”回退

## 9. 测试与验证

本项目为纯静态站，本次验证包括：

- 本地静态服务下首页加载正常
- 桌面端截图检查视觉结构
- 移动端截图检查首屏与卡片排布
- 确认无横向滚动条
- 确认所有主入口卡片可点击
- 确认移除文章筛选后控制台无脚本错误

## 10. 实施边界

本次改版只处理首页，目标是尽快将站点从“博客首页”切换成“个人名片入口页”。

后续可选扩展，但不属于本次范围：

- 真正的多语言切换
- 真实头像与社交链接接入
- 邮件订阅接口
- 深色 / 浅色主题切换
- 项目详情页与作品集页

## 11. 最终预期

改版完成后，首页应呈现为：

- 一个以个人身份为中心的品牌入口页
- 一个更贴近男性开发者 / AI 从业者气质的视觉版本
- 一个移动端可直接分享的链接名片
- 一个后续只需替换 mock 内容即可继续使用的静态页面
